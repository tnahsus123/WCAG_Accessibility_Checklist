# Contributing

Thank you for your interest in contributing! 🎉

This project aims to make web accessibility resources more accurate, practical, and accessible for everyone.

Whether you're fixing a typo, improving documentation, adding examples, or reporting an issue, your contributions are welcome.

---

## Ways to Contribute

You can contribute by:

- Fixing grammar, spelling, or formatting
- Improving explanations
- Correcting inaccurate information
- Adding practical accessibility examples
- Adding WCAG 2.2 resources
- Improving checklists
- Updating links to official resources
- Reporting bugs or documentation issues
- Suggesting new topics

---

## Before You Start

Please:

- Search existing Issues before creating a new one.
- Keep changes focused and relevant.
- Follow the existing formatting and writing style.
- Prefer official WCAG and W3C references where applicable.

---

## Contribution Process

1. Fork the repository.
2. Create a new branch.

```
git checkout -b feature/your-feature-name
```

3. Make your changes.
4. Commit your work.

```
git commit -m "Add keyboard navigation examples"
```

5. Push your branch.

```
git push origin feature/your-feature-name
```

6. Open a Pull Request describing your changes.

---

## Writing Guidelines

Please ensure that documentation is:

- Clear and beginner-friendly
- Technically accurate
- Well-structured with headings
- Written in plain language
- Free of unnecessary jargon
- Accessible and inclusive

---

## Accessibility Guidelines

When contributing examples:

- Use semantic HTML whenever possible.
- Avoid unnecessary ARIA.
- Include accessible code examples.
- Follow WCAG 2.2 Level AA recommendations.

---

## Reporting Issues

When opening an issue, please include:

- A clear title
- A description of the problem
- Steps to reproduce (if applicable)
- Suggested improvements (optional)

---

## Code of Conduct

Please be respectful and constructive.

We welcome contributors from all backgrounds and are committed to maintaining a welcoming and inclusive community.

---

## Thank You

Your contributions help improve accessibility knowledge for developers, designers, students, and organizations around the world.

Thank you for making the web more accessible! ❤️
